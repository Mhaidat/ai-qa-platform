# User Journey

## MVP End-to-End Flow

1. User registers.
2. System creates an Organization.
3. First user becomes Organization Owner/Admin.
4. Admin invites team members.
5. Admin assigns roles.
6. Admin creates a Project.
7. Admin connects Jira.
8. User imports Stories from Jira or creates Stories manually.
9. System stores Stories under the Project.
10. System builds initial Knowledge Base from Stories.
11. Business Analyst runs AI Requirement Analysis.
12. Business Analyst reviews gaps, ambiguities, business rules, and conflicts.
13. Business Analyst updates and approves the Story.
14. Manual QA generates test cases from the approved Story.
15. Manual QA reviews, edits, deletes, or adds manual test cases.
16. Manual QA approves test cases.
17. Manual QA syncs or links approved test cases with Jira.
18. Automation QA selects approved test cases.
19. Automation QA generates Playwright TypeScript code.
20. Automation QA reviews and downloads the generated automation project.
